# Roll a Ball

## Short Description

**Roll a Ball** is a 3D gameplay prototype developed in **Unity** as part of the Programming II course of the University Technical Degree in Game Design and Production (UTN).

The project goes beyond the classic Unity tutorial by focusing on clean architecture, modular gameplay systems, and Object-Oriented Programming principles. It was designed to explore player interaction, enemy behaviors, reusable gameplay components, and scalable game architecture while implementing software engineering concepts commonly used in professional game development.

---

## Overview

The game features a complete gameplay loop including:

- Third-person player controller
- Modular enemy system with movement, detection and attack behaviors
- Stomp mechanic with collision validation
- Event-driven UI feedback
- Scene-based game flow (Main Menu, Gameplay and Game Over)
- Victory and defeat conditions

## Technical Highlights

This project was developed with a strong focus on maintainable and scalable architecture.

Highlights include:

- Separation of responsibilities between input, movement, combat, enemies and UI.
- Reusable enemy base class extended through inheritance and polymorphism.
- Interface-driven gameplay systems for damage and stomp interactions.
- Modular gameplay architecture designed for future expansion.
- Event-based communication between gameplay and UI.
- Coroutine-based state and timing management.
- Structured collision handling using reusable utility classes.
- Scene management with complete gameplay flow.

Rather than only producing a functional prototype, the project emphasizes software engineering practices applicable to larger Unity productions.

## Lessons

This project significantly strengthened my understanding of both Unity development and software architecture.

Key takeaways include:

- Designing gameplay systems using Object-Oriented Programming.
- Applying inheritance, interfaces and polymorphism to reduce code duplication.
- Implementing common game programming design patterns.
- Building modular gameplay systems that can scale beyond a single prototype.
- Managing interactions between multiple gameplay systems.
- Working with C# collections to organize gameplay data efficiently.
- Debugging complex physics and collision behaviors.
- Structuring Unity projects for long-term maintainability.
- Understanding the importance of clean architecture over simply achieving functionality.

This project represents an important step in transitioning from tutorial-based development toward building maintainable, production-oriented Unity projects.
